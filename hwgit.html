What is GIT?

GIT is an open source VCS that stores data as a stream of snapshots. When there are multiple developers working on the same project, Git ensures there are no code conflicts, allows developers to revert to old versions of the code, and has branches so developers can work in parallel to each other. There are three main sections of a Git project: the working directory, the staging area, and the repository. The basic workflow starts with modifying files, staging the files, then committing them, which adds them to the local repository.

Why do we need source code Version Control System?

Source code Version Control Systems records changes to files so that users can recall specific versions at a later time. Graphic and web designers especially find this useful as it allows the user to see who else modified the file, who created an issue, and allows reversion back to a previous state.
